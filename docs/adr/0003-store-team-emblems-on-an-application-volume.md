# Store Team Emblems on an application volume

ESPN remains an ingestion source for Match data and Team Emblems, but the application will download each discovered Team Emblem and serve its retained copy rather than exposing ESPN URLs to browsers. Each Team owns one reusable Team Emblem; it is persisted in a Railway Volume in production and a Git-ignored local directory in development, with the database storing its metadata and stable application path. Changed emblems receive a new content-hashed path; their old files become cleanup candidates rather than being deleted immediately. The server owns the dedicated cleanup module because it owns the mounted Volume; a separate Railway cron service triggers it daily through Railway's private network and a secret token. A failed refresh preserves the last known-good Team Emblem so ESPN outages do not affect Match pages; a new Team whose initial download fails remains available with a neutral browser fallback and is retried on later scrapes.
